ВКонтакте API

Для работы с API ВКонтакте необходимо:

  • Создать приложение на странице разработчика ВКонтакте
  • Получить токен доступа с правами на работу со стеной и группами
  • Указать токен в настройках приложения

Документация API:

  • wall.get - метод для получения записей со стены
  • groups.getById - метод для получения информации о группах
Одноклассники API

Для работы с API Одноклассники необходимо:

  • Создать приложение на странице разработчика Одноклассники
  • Получить публичный и секретный ключи приложения
  • Получить токен доступа для работы с API

Документация API:

  • stream.get - метод для получения потока активности
  • mediatopic.get - метод для получения записей из группы
О системе
Система мониторинга упоминаний в социальных сетях

Данное приложение позволяет отслеживать упоминания в сообществах социальных сетей ВКонтакте и Одноклассники по заданным ключевым словам.

Основные возможности:
  • Мониторинг упоминаний в сообществах ВКонтакте и Одноклассники
  • Поиск по ключевым словам
  • Выделение найденных ключевых слов цветом
  • Экспорт результатов в CSV
  • Управление несколькими проектами (до 5)
  • Фоновый автоматический поиск
  • Система логирования
Технические детали:
  • Фреймворк: Flask (Python)
  • База данных: SQLite
  • Интерфейс: Bootstrap 5
  • Фоновые задачи: Python Threading
Советы по использованию

Для эффективного мониторинга упоминаний используйте следующие подходы:

  • Используйте разные формы слова (например, "товар", "товары", "товаром")
  • Добавляйте часто встречающиеся опечатки
  • Включайте имена конкурентов и название вашей компании
  • Используйте профессиональные термины, связанные с вашей отраслью

Для оптимальной работы фонового поиска:

  • Не добавляйте слишком много сообществ для одновременного мониторинга (оптимально до 10-15)
  • Имейте в виду ограничения API социальных сетей по количеству запросов
  • При большом количестве результатов рекомендуется экспортировать их регулярно, чтобы снизить нагрузку на систему

Эффективное использование нескольких проектов:

  • Создавайте отдельные проекты для разных клиентов или направлений бизнеса
  • Используйте разные API токены для разных проектов, чтобы не превышать лимиты запросов
  • Регулярно экспортируйте данные отдельно по каждому проекту
  • Используйте понятные названия проектов для удобной навигации
